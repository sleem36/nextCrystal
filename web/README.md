# Crystal Motors MVP

Премиальная посадка для ускоренной воронки:

`Hero CTA -> Быстрый подбор -> Релевантные авто -> Заявка (e-mail)`.

## Шаг 1. План, риски, оценка

### Этапы разработки

1. Базовый каркас Next.js + Tailwind + архитектура компонентов.
2. Стиль-гайд и унификация UI-токенов.
3. Сборка MVP-экранов и сценария воронки.
4. Серверная отправка лидов на e-mail.
5. Подключение Яндекс.Метрики и целей.
6. Ручная проверка и стабилизация.

### Риски

- Некорректные SMTP-параметры могут ломать отправку заявок.
- Без CRM заявки только по e-mail (нужен контроль доставки).
- Необходима внимательная проверка mobile UX и текстов оффера.

### Оценка для 1 разработчика

- Базовая реализация MVP: `1-2 дня`.
- Полировка контента/состояний/трекинга: `0.5-1 день`.

## Шаг 2. Стиль-гайд

Страница: `/style-guide`

- Цвета: `primary`, `neutral`, `success`, `warning`.
- Типографика: уровни заголовков, основной и вспомогательный текст.
- Компоненты: кнопки, инпуты, карточки, бейджи.
- Состояния: `hover/focus/disabled/error`.
- Принцип: чистая сетка, контрастные CTA, минимум визуального шума.

## Шаг 3. Что реализовано в MVP

- Главная посадка: `hero + 1 главный CTA + блок доверия`.
- Быстрый подбор по бюджету и сценарию.
- Короткая выдача релевантных авто.
- Карточка "решение за 5 секунд" (цена, платеж/мес, пробег, доверие, CTA).
- Форма заявки с обязательным телефоном.
- API отправки лида на e-mail через SMTP.
- Передача контекста: `ID авто`, `город`, `бюджет`, `UTM`.
- Яндекс.Метрика + цели.

## Шаг 4. Запуск и ручная проверка

### Установка и запуск

```bash
npm install
npm run dev
```

Сайт: [http://localhost:3000](http://localhost:3000)  
Style-guide: [http://localhost:3000/style-guide](http://localhost:3000/style-guide)

### Переменные окружения

Создайте `.env.local`:

```env
NEXT_PUBLIC_YANDEX_METRIKA_ID=00000000
LEADS_TO_EMAIL=sales@example.com
SMTP_HOST=smtp.example.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=bot@example.com
SMTP_PASS=secret
SMTP_FROM=bot@example.com
CARS_API_URL=
```

### Проверка вручную

1. На первом экране один приоритетный CTA.
2. CTA скроллит к блоку быстрого подбора.
3. Подбор фильтрует карточки по параметрам.
4. Выбор карточки скроллит к форме.
5. Отправка с валидным телефоном возвращает успешный статус.
6. В письме есть контекст (`selectedCarId`, `monthlyBudget`, `utm_*`).
7. Цели Метрики срабатывают на CTA/подбор/заявку.
8. В футере открываются legal-страницы.
9. Антиспам блокирует слишком быстрые/частые отправки.

### Что осталось (после MVP)

- Подключить реальный источник авто вместо локального массива.
- Добавить legal-страницы (политика, согласие).
- Подготовить A/B офферы под рекламные сегменты.
