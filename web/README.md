# Crystal Motors MVP

Лендинг и мини-каталог для подбора подержанных автомобилей:

`Главная -> Квиз -> Каталог -> Карточка авто -> Заявка`.

## Шаг 1. План, риски, оценка

### Этапы разработки

1. Базовый каркас Next.js + Tailwind + архитектура компонентов.
2. Стиль-гайд и унификация UI-токенов.
3. Сборка MVP-экранов и сценария воронки.
4. Серверная отправка лидов на e-mail.
5. Подключение Яндекс.Метрики и целей.
6. Ручная проверка и стабилизация.

### Риски

- Некорректные SMTP-параметры могут ломать отправку заявок.
- Без CRM заявки только по e-mail (нужен контроль доставки).
- Необходима внимательная проверка mobile UX и текстов оффера.

### Оценка для 1 разработчика

- Базовая реализация MVP: `1-2 дня`.
- Полировка контента/состояний/трекинга: `0.5-1 день`.

## Шаг 2. Стиль-гайд

Страница: `/style-guide`

- Цвета: `primary`, `neutral`, `success`, `warning`.
- Типографика: уровни заголовков, основной и вспомогательный текст.
- Компоненты: кнопки, инпуты, карточки, бейджи.
- Состояния: `hover/focus/disabled/error`.
- Принцип: чистая сетка, контрастные CTA, минимум визуального шума.

## Шаг 3. Что реализовано в MVP

- Главная посадка: `hero + 1 главный CTA + блок доверия`.
- Быстрый подбор по 5 шагам квиза (без дополнительного выбора сценария).
- Короткая выдача релевантных авто.
- Карточка "решение за 5 секунд" (цена, платеж/мес, пробег, доверие, CTA).
- Форма заявки с обязательным телефоном.
- API отправки лида на e-mail через SMTP.
- Передача контекста: `ID авто`, `город`, `бюджет`, `UTM`.
- Яндекс.Метрика + цели.

## Шаг 4. Запуск и ручная проверка

### Установка и запуск

```bash
npm install
npm run dev
```

Сайт: [http://localhost:3000](http://localhost:3000)  
Style-guide: [http://localhost:3000/style-guide](http://localhost:3000/style-guide)

### Переменные окружения

Создайте `.env.local`:

```env
NEXT_PUBLIC_YANDEX_METRIKA_ID=00000000
LEADS_TO_EMAIL=sales@example.com
SMTP_HOST=smtp.example.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=bot@example.com
SMTP_PASS=secret
SMTP_FROM=bot@example.com
CARS_API_URL=
```

### Роуты

- `/` — главная: hero, квиз, базовая выдача.
- `/cars` — каталог с фильтрами по query.
- `/cars/[id]` — карточка автомобиля.
- `/compare` — сравнение авто по `?ids=id1,id2,id3`.
- `/api/cars` и `/api/cars/[id]` — mock API каталога.
- `/api/lead` — приём заявки.

### Проверка вручную

1. На первом экране один приоритетный CTA; доверие/дисклеймеры — в аккордеоне «Подробнее».
2. CTA раскрывает блок подбора на месте (без автоскролла к якорю).
3. Квиз из 5 шагов в одном блоке; выдача появляется под квизом без принудительного скролла.
4. Клик по карточке авто открывает модалку заявки (без перехода к форме внизу страницы).
5. Отправка с валидным телефоном возвращает успешный статус.
6. В письме есть контекст (`carId`, `city`, `monthlyBudget`, `maxPriceRub`, `bodyType`, `transmission`, `purchaseGoal`, `utm_*`).
7. Цели Метрики: `hero_cta_click`, `quiz_completed`, `catalog_from_quiz_click` (переход в каталог с параметрами квиза), `lead_modal_open` (открытие модалки), `lead_submitted` (отправка формы).
8. В футере открываются legal-страницы.
9. Антиспам блокирует слишком быстрые/частые отправки.

В Метрике создайте цель `lead_modal_open`, если используете воронку с модалкой заявки.

### E2E smoke (pre-release)

- Запуск одной командой: `npm run e2e:smoke`
- Что проверяется:
  - `/` открывается и виден главный CTA.
  - Квиз на главной -> переход в `/cars` с query фильтрами.
  - `/cars` -> переход в `/cars/[id]`, на странице есть блок заявки.
  - `/compare?ids=...` открывается и показывает таблицу.
  - `/cars/not-found-id` отдаёт 404-экран.
  - API: `/api/cars/[valid]` -> 200, `/api/cars/[invalid]` -> 404.
- Тесты не требуют SMTP: отправка лида не выполняется, проверяется UI/API.

### Квиз и каталог (`/cars`)

- Ответы квиза сохраняются в `localStorage` под ключом **`crystal_quiz_answers`** после нажатия «Показать подходящие варианты».
- Кнопка **«Смотреть подборку в каталоге»** ведёт на `/cars` с теми же query-параметрами, что и фильтры листинга; UTM с главной переносятся в ссылку.
- На `/cars` без фильтров в URL, если есть сохранённый квиз, показывается баннер **«Применить из квиза»** (без автоподстановки без клика). «Скрыть» помечает сессию: баннер не показывается до нового прохождения квиза (ключ `sessionStorage`: `crystal_quiz_catalog_banner_dismissed`).
- Чтобы сбросить сохранённые ответы квиза вручную: очистите `localStorage` для сайта или удалите ключ `crystal_quiz_answers` в DevTools.

### LocalStorage ключи

- `crystal_quiz_answers` — сохранённые ответы квиза.
- `crystal_compare_car_ids` — выбранные авто для сравнения (до 3).
- `crystal_quiz_catalog_banner_dismissed` (sessionStorage) — скрытие баннера «Применить из квиза» до нового прохождения квиза.

### Связка квиз -> каталог -> лид

1. На главной пройти квиз и нажать «Смотреть подборку в каталоге».
2. В `/cars` проверить, что фильтры выставлены через query (параметры `monthlyBudget`, `maxPriceRub`, `bodyType`, `transmission`, `city`, ...).
3. Открыть `/cars/[id]`, отправить заявку и проверить контекст в теле запроса/письма (`carId`, фильтры, `utm_*`).
